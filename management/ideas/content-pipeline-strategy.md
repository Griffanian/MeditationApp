# Content Pipeline: B2B to B2C Flywheel

## The Loop
1. **Teachers use the builder tools** to create and structure practices for their students. They're not doing extra work — the builder is just the best tool for the job.
2. **The platform accumulates a library** of tradition-specific, expert-created content that's already tagged and structured because the builder encouraged that from the start.
3. **With teacher consent**, that content gets curated into B2C skill tree progression paths — real content made by real practitioners, not generic filler.
4. **Practitioners discover teachers** through the skill trees, feeding students back to teachers who want them.

Teachers get distribution. Practitioners get structured paths from quality content. The platform gets both sides feeding each other.

## The Key Move: Metadata Now, Skill Trees Later
Design the builder tools so they capture the right metadata from day one, even before skill trees exist:
- What tradition is this practice from?
- What does it train? (e.g. breath awareness, loving-kindness, proprioception)
- What level is it appropriate for? (beginner / intermediate / advanced)
- Any prerequisites?

This metadata serves the teacher immediately (organising their own content library) and serves the platform later (slotting content into skill trees). Teachers aren't filling out metadata "for you" — they're doing it because it helps them manage their own stuff.

## Sequencing
1. **Now:** Build the core — editor, practice builder, content management. Already in talks with a company to white-label the platform as a feature within their app, with their people creating meditations using the builder tools.
2. **First traction:** White label + B2B. The partner company is the first customer — their creators use the builder, their app serves the content.
3. **Parallel / ongoing:** More B2B relationships with independent teachers.
4. **When the library is deep enough:** Layer on B2C skill trees using the content that's already been created through the B2B/white-label channels.

## Consent & Ownership
- Teachers must explicitly consent to their content being used in platform-curated skill trees.
- Need to figure out the right model here — licensing, revenue share, attribution, ability to opt out at any time.
- This is a trust relationship. If teachers feel like the platform is extracting their content, the flywheel breaks.
